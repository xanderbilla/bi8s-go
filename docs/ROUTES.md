# Routes

This document is the canonical, hand-curated index of every HTTP route the
binary exposes. It is generated by reading [`app/internal/http/router.go`](../app/internal/http/router.go)
and the per-handler `Register` functions; treat the source as the source of
truth and update this file whenever a route is added, removed, or renamed.

For the machine-readable contract see [`openapi.yaml`](openapi.yaml).
For the response envelope and error shape see [`API.md`](API.md).

## Sections

- Public consumer routes (`/v1/c/*`)
- Admin routes (`/v1/a/*`) — protected via the auth-stub middleware in
  [`app/internal/http/middleware/authstub`](../app/internal/http/middleware/authstub/authstub.go);
  swap in real auth without changing the route table.
- Operational routes (`/healthz`, `/readyz`, `/metrics`, `/v1/openapi.yaml`)
- Static / streaming routes (HLS, image proxy, signed-URL redirects)

## How to extend

1. Add the handler under `app/internal/http/`.
2. Register it via the appropriate router group in `router.go`.
3. Update `openapi.yaml` and re-run `make openapi-lint`.
4. Append the new route to the table above with method, path, auth tier,
   rate-limit class, and a one-line description.
